✅ ToDoList: Interactividad avanzada del prefijo telefónico

🔹 1. Estado y modelo de datos

  • Separar en el estado el prefijo (paisTelefono.codigo) y la bandera (paisTelefono.bandera).

  • Guardar ambos juntos en un objeto, por ejemplo: 

    js
    paisTelefono: { codigo: '+51', bandera: 'https://flagcdn.com/pe.svg' }

🔹 2. Input editable para el prefijo

  • Hacer que el prefijo (+51) no sea fijo, sino un input editable.

  • Permitir solo caracteres + y números (regex: /^\+?\d*$/).

  • Detectar cambios en tiempo real (onChange) y:

    • Buscar si el código coincide con algún país en la lista (countriesData).

    • Si hay match → actualizar la bandera y normalizar el código.

    • Si no hay match → mostrar bandera neutral o gris (por ejemplo 🌐 o 🇺🇳).

🧠 Tip UX: podríamos usar un debounce (300 ms) para no buscar en cada tecla.

🔹 3. Auto-detección del país

  • Tener un array countriesData con { codigo: '+51', iso2: 'pe', bandera: 'https://flagcdn.com/pe.svg', pais: 'Perú' }.

  • Función auxiliar: 

    js
    const getCountryByCode = (code) =>
  countriesData.find(c => code.startsWith(c.codigo));

  • Si el usuario escribe +34, la función devuelve España, actualizamos bandera 🇪🇸.

🔹 4. Sincronización visual

  • Mostrar la bandera siempre alineada al prefijo, incluso cuando el prefijo se escribe manualmente.

  • Si el prefijo no coincide con ningún país conocido → mostrar icono genérico 🌐 o sin bandera.

  • El input del número (telefono) debe mantenerse separado del prefijo, para no reescribir accidentalmente.

🔹 5. Integración con el dropdown existente

  • Si el usuario abre el dropdown, puede elegir un país como hasta ahora.

  • Esa selección:

    • Sobrescribe el código actual.

    • Actualiza también la bandera.

  • Si luego el usuario edita el prefijo manualmente, se debe reflejar de nuevo la bandera sin interferir.

🔹 6. Validaciones UX extra

  • Si el usuario borra el “+”, el sistema debería reinsertarlo automáticamente al inicio.

  • Prevenir espacios en blanco.

  • Si el código es inválido (no existe), mostrar feedback visual (por ejemplo, color gris tenue o tooltip "prefijo no reconocido").

🔹 7. Opcional: enriquecimientos futuros

  • Sugerencias automáticas mientras se escribe el prefijo (+1 → mostrar lista de países con +1: US, Canadá...).

  • Autocompletar país si se selecciona un código compartido.

  • Integrar librería externa tipo libphonenumber-js para validaciones más reales.

🔹 8. Ejemplo de flujo final esperado

Escribe +54 manualmente Prefijo cambia → bandera actualiza a 🇦🇷
